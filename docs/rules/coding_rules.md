# 宿泊税システム コーディング規約

## 1. 概要
- **使用言語:** Java 21
- **フレームワーク:** Spring Boot
- **目的:** 品質・生産性・保守性の向上。規約外の手法の利用は原則禁止。

## 2. ディレクトリ構成
- `controllers/`: APIエンドポイント
- `dto/`: データ転送用モデル
- `entities/`: DBデータモデル
- `services/`: ビジネスロジック
- `repositories/`: DBアクセス
- `utils/`: 共通ユーティリティ

## 3. 命名規則
- **クラス名:** アッパーキャメルケース（例: `UserController`）
- **メソッド名/変数名:** ローワーキャメルケース（例: `getUserInfo`）
- **パッケージ名:** 全小文字
- **HTMLファイル:** ローワーキャメルケース（例: `userList.html`）
- **定数名:** 全大文字アンダースコア区切り（例: `MAX_RETRY_COUNT`）

## 4. インジェクション規約
- **原則:** フィールドインジェクション（`@Autowired`）は禁止。
- **手法:** **コンストラクタインジェクション**を必須とする。
- **実装:** `Lombok` の `@RequiredArgsConstructor` を使用すること。

## 5. Lombok利用ルール
- `@Data`: Entityクラスで3つ以上の機能を付与する場合に使用。
- `@Getter` / `@Setter`: 必要に応じてクラス単位で付与。
- `@RequiredArgsConstructor`: コンストラクタインジェクション用。

## 6. コーディングスタイル
- **インデント/行の長さ:** Eclipseの自動フォーマット（Ctrl+Shift+F）に従う。
- **括弧:** 開き括弧 `{` は行末、閉じ括弧 `}` は独立した行に置く。
- **マジックナンバー:** 直接の数値記述は禁止。定数クラスで定義すること。
